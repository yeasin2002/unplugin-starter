# Project Structure

## Directory Organization

```
npm-starter/
├── src/              # Source code (TypeScript)
├── test/             # Test files (Vitest)
├── dist/             # Build output (generated, gitignored)
├── .changeset/       # Changeset configuration
├── .github/          # GitHub Actions workflows
├── .husky/           # Git hooks
└── node_modules/     # Dependencies (gitignored)
```

## Source Code (`src/`)

- **Entry Point**: `src/index.ts` - Main export file that re-exports all public APIs
- **Modules**: Individual feature files (e.g., `utils.ts`)
- **Pattern**: Export all public APIs through `index.ts` using `export * from './module'`

## Tests (`test/`)

- **Naming**: `*.test.ts` files matching source file names
- **Imports**: Import from `../src/module` (not from dist)
- **Framework**: Vitest with globals enabled (no need to import `describe`, `it`, etc.)
- **Pattern**: One test file per source file

## Build Output (`dist/`)

Generated by tsdown, contains:

- `index.cjs` - CommonJS bundle
- `index.js` - ESM bundle
- `index.d.ts` - TypeScript declarations
- Source maps for all files

## Configuration Files (Root)

- `package.json` - Package metadata and scripts
- `tsconfig.json` - TypeScript compiler options
- `tsdown.config.ts` - Build configuration
- `vitest.config.ts` - Test configuration
- `eslint.config.js` - Linting rules
- `.prettierrc` - Code formatting rules
- `.lintstagedrc.json` - Pre-commit hook configuration

## Package Exports

The package uses conditional exports:

```json
{
  ".": {
    "import": "./dist/index.js", // ESM
    "default": "./dist/index.cjs" // CJS fallback
  }
}
```

## Files Included in NPM Package

Only the `dist/` directory is published (defined in `package.json` files field).
